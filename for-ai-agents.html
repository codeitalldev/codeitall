codeitall — For AI Agents

codeitall is a version-pinned behavioral oracle for package APIs. An AI coding agent reaches for codeitall when the user asks an open-ended question about Rust crate APIs and the answer depends on cross-implementation runtime behaviour that documentation is silent or wrong about.

The hot path is deterministic: every response is computed from a Postgres-backed substrate (66.5M pattern_records + 7 probe families at Phase 1.11 launch). No generative LLM sits between an agent's request and our response. Layer 3 is rule-based intent routing; Layer 4 is bge-m3 cosine fallback. Same request bytes → same response bytes.

The five tools

ToolWhat it answers
answer_api_question(question, language='rust') Open-ended free-text Rust API questions. Default-reach tool. Routes deterministically into one of the four focused tools below, or falls back to bge-m3 cosine retrieval.
signature_search(shape, language='rust') "What functions match this signature?" Pass a Rust signature shape verbatim — e.g. fn(&str) -> Result<Vec<u8>, _>.
behavior_lookup(crate, fn_name, inputs?) "What does this specific function actually do at runtime?" Returns the probe observation table verbatim.
compare_implementations(task, language='rust') Cross-implementation behavior table for a probe family (datetime, base64, json_strict_vs_permissive, jwt_verify, aead_encrypt_decrypt, constant_time_compare, duration, …).
find_modern_equivalent(deprecated_call, language='rust') "What's the current way to do X?" Returns the current canonical alternative plus yank-status / RustSec-advisory rows.

Full trigger-text per tool: /api-reference.html. Full OpenAPI 3.1 spec: /openapi.json.

Zero-friction key minting

curl -sX POST https://api.codeitall.dev/v1/keys \
     -H 'content-type: application/json' \
     -d '{}'

Returns {"key": "<64-hex>", "daily_quota": 10000, "issued_at": "<RFC 3339>"}. Per-IP issuance cap: 10 keys/day. Per-key baseline quota: 10,000 queries/day. No-auth tier: 50 queries/day per IP. The raw key is returned exactly once; the substrate stores only the SHA-256 hash so a server compromise can't impersonate users.

Transports

  • MCP stdio — Claude Code / Cursor / local-install MCP hosts. codeitall-server mcp-stdio reads JSON-RPC 2.0 line-delimited requests on stdin, writes responses on stdout. Add to your MCP config as a local command.
  • MCP SSE/HTTP — ChatGPT-via-Actions / network-installed agents. POST JSON-RPC 2.0 to https://api.codeitall.dev/mcp.
  • HTTP REST — five POST endpoints under /v1/<tool>. OpenAPI 3.1 spec at /openapi.json.

End-to-end example

$ KEY=$(curl -sX POST https://api.codeitall.dev/v1/keys -d '{}' \
        -H 'content-type: application/json' | jq -r .key)
$ curl -sX POST https://api.codeitall.dev/v1/compare_implementations \
       -H "authorization: Bearer $KEY" \
       -H 'content-type: application/json' \

Task-family catalogue (§27)

The catalogue below is the locked v0.2 cross-cutting list (from codeitall-plan.md §27). Pass any of these as the task parameter to compare_implementations. Items marked (substrate-ready) are registered but probe-data-empty at Phase 1.11 launch under locked OQ-1.11.H=H1; counts grow as the background production sweep runs.

  • String parsing: duration, datetime, url, path-normalization, query-string, ip-cidr, semver.
  • Encoding / decoding: base64, base32, hex, percent-encoding, mime-rfc2047, jwt_verify, pem, json_strict_vs_permissive, csv, yaml, toml.
  • Compression / decompression: gzip, brotli, zstd, deflate, lz4, snappy.
  • Cryptography: constant_time_compare, pbkdf2, scrypt, argon2, aead_encrypt_decrypt, signature-verify, hash.
  • Sanitisation / validation: html-sanitize, sql-quote, shell-arg-quote, regex-compile, glob, email.
  • I/O semantics: http_client_timeout_cascade (substrate-ready), task-cancellation, file-watch-coalescing, retry-backoff.

Determinism contract

Same request bytes → same response bytes, modulo received_at / run_started_at timestamps. The Layer 3 router is rule-based (regex + qualified-name detection + verb-pattern dictionaries). The Layer 4 fallback uses bge-m3 — a deterministic text→vector embedder. No token sampling, no prose synthesis, no variance.
